# tests/snippets_pix3eq_workflow.txt — pix3eq + ShotCap workflow example
#
# This file is a template. Replace the placeholder pix3eq line in
# tFP_check below with the line ShotCap -inspect copied to your
# clipboard, then run cf22 against this file.
#
# Step-by-step:
#
# 1. Capture a screenshot showing the glyph/region you want to detect:
#       ShotCap\x64\Debug\ShotCap.exe -w "ColorForth" -f cf22.png
#    (or any other capture flags — see -h)
#
# 2. Open the capture in the inspector:
#       ShotCap\x64\Debug\ShotCap.exe -inspect cf22.png
#
# 3. Inspector window:
#       - Mouse or arrow keys move the yellow crosshair
#       - Shift+Arrow = 10 px steps
#       - +/- zoom in/out (default 8x)
#       - Click or press Space to record current pixel as a sample
#       - After the 3rd sample, a line is printed AND copied to clipboard:
#           0x00000183 0x00000083 0x00008CFF 0x00000119 0x00000138 0x0000F708 0x00000061 0x0000024D 0x00F70000 10 pix3eq
#       - R resets samples, ESC closes
#
# 4. Replace the placeholder line below with the clipboard contents.
#
# 5. Launch cf22:
#       color_iw.exe tests\snippets_pix3eq_workflow.txt
#
# 6. Make sure the screen looks the same as when you captured (same window
#    position, same scene). Press Ctrl+Shift+1.
#
# 7. Open color_debug.log. Look for:
#       [PIX3EQ] result=00000001   → fingerprint matched (scene unchanged)
#       [PIX3EQ] result=00000000   → fingerprint missed (something changed)
#
# Notes:
#   - Tolerance is the LAST value before `pix3eq` (10 = ±10 per channel).
#     Raise it (e.g. 20) if matches are too tight, lower for stricter.
#   - Coordinates are absolute SCREEN coordinates (from desktop GetPixel).
#     They must match where the captured pixels are on the live screen.
#   - All 3 fingerprints must match → AND-semantics, so one mismatch fails.
