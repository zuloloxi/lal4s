# tests/snippets_pix3eq.txt — pix3eq smoke test
#
# pix3eq ( p1x p1y col1 p2x p2y col2 p3x p3y col3 tol -- 0|1 )
#   Reads 3 absolute desktop pixels via GetPixel and AND-checks each
#   against its expected COLORREF within per-channel `tol`. Pushes
#   1 if all 3 match, 0 otherwise. This is the CocBot v5 OCR atom:
#   a glyph-fingerprint table feeds this primitive per candidate
#   character.
#
# COLORREF format Win32 = 0x00BBGGRR.
#   pure red    = 0x0000FF
#   pure green  = 0x00FF00
#   pure blue   = 0xFF0000
#   pure white  = 0xFFFFFF
#
# Verification — pix3eq emits one log line per call:
#   [PIX3EQ] result=00000001    → all 3 matched
#   [PIX3EQ] result=00000000    → at least one missed
#
# How to test:
#   1. Position cf22 so its blue background covers the sample region
#      (~500, 400). Same area as snippets_pixel_avg.txt sampled.
#   2. Ctrl+Shift+1 → all 3 pixels asked for "blue" → result=1
#   3. Ctrl+Shift+2 → ask for "red" → result=0 (cf22 isn't red there)
#   4. Ctrl+Shift+3 → mix: 2 pixels match blue, 1 pixel asks for red
#                     → result=0 (AND-rule)
#   5. Ctrl+Shift+5 → flash a debug_box at the sample region so you
#                     can see WHERE we're checking.

::: tP3_show_box ! Ctrl+Shift+5
470 380 530 420 1000 debug_box

::: tP3_blue_match ! Ctrl+Shift+1
# Three pixels inside cf22's blue area, all asking for blue.
# Expect: result=00000001
"pix3eq_blue_match" tname
500 400 0xFF0000  502 400 0xFF0000  500 402 0xFF0000  20 pix3eq
tsummary

::: tP3_red_miss ! Ctrl+Shift+2
# Three pixels in same blue area, all asking for red.
# Expect: result=00000000
"pix3eq_red_miss" tname
500 400 0xFF  502 400 0xFF  500 402 0xFF  20 pix3eq
tsummary

::: tP3_mixed_miss ! Ctrl+Shift+3
# Two pixels ask for blue (match), one asks for red (miss).
# Demonstrates AND semantics: even one miss → result=0.
"pix3eq_mixed_miss" tname
500 400 0xFF0000  502 400 0xFF0000  500 402 0xFF  20 pix3eq
tsummary
